## What are HTML forms, and why would we use them?

You've probably entered data into a form before for one of many reasons:

- To sign up or sign in to a website.
- To submit order details, such as credit card or address information.
- To deliver feedback about a product or service.
- To conduct a search for information.

Forms enable all of this, and so much more! Submitting a form and clicking a link are the only two user actions that can generate an HTTP request on a webpage without utilizing JavaScript, so they are foundational to providing user interactivity on the internet.

Because they are such a heavy driver of user interaction and data collection, forms are one of the most vital parts of a webpage. Developers have many tools available to them to help ensure the data collected from forms is reliable and that they can be naviagated easily:

- Different input methods such as text input, dropdowns, checkboxes, date inputs, and more.
- Accessiblity enhancements like `Tab` focus (hitting the `Tab` key to move from one field to another)
- Validation messages to ensure users are providing data correctly - for example, checking that an email address is formatted like an email address.
- So much more!
